## Bugs Found

### Bug #1: Document Create — parentId Validation Error
- **嚴重度**: HIGH
- **位置**: `POST /api/documents`
- **行為**: 點擊「新增文件」→ 輸入標題 → alert 顯示 `{"error":"輸入驗證失敗","fields":{"parentId":["Expected string, received null"]}}`
- **原因**: 前端送 `parentId: null`，但 Zod schema 要求 string 型別
- **影響**: 無法建立任何新文件
- **修復建議**: API schema 改為 `z.string().nullable()` 或前端在無 parent 時送空字串/省略欄位

### Bug #2: Time Entry Manual Create — API 400
- **嚴重度**: HIGH
- **位置**: `POST /api/time-entries`
- **行為**: 工時格子點 `+` → 填入 2.5h → 儲存 → 400 Bad Request
- **原因**: 待查（可能是 Zod validation 或 taskId 格式問題）
- **影響**: 手動輸入工時功能失效（Timer 自動建立正常）
- **註**: Timer 停止後自動建立工時是正常的，只有手動填入格子失敗

### Bug #3: Notification Preferences — API 400
- **嚴重度**: MEDIUM
- **位置**: `PUT /api/users/admin-001/notification-preferences`
- **行為**: 點擊通知 toggle → API 返回非 200 → toggle 未生效
- **影響**: 無法變更通知偏好設定
