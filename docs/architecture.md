## 2. 連接埠分配 (Port Assignments)

| 連接埠 | 協定 | 對應服務 | 說明 |

## 4. 資料流 (Data Flow)

### 4.1 使用者登入流程

```
[使用者瀏覽器]
     │
     │ HTTPS:443
     ▼
[Nginx 反向代理]
     │ 路由至對應服務（Outline/Plane 等）
     ▼
[應用服務容器]
     │
     ├──→ [Redis] 快取 Session Token
     │
     └──→ [AD/LDAP Server]（TCP:636）
               │ 驗證使用者憑證
               ▼
          [回傳使用者屬性：群組、Email]
               │
               ▼
     [應用服務建立 Session]
               │
               ▼
     [PostgreSQL 儲存使用者資料]
```

### 4.2 文件上傳流程（Outline）

```
[使用者上傳附件]
     │
     ▼
[Outline 容器（:3000）]
     │
     ├── 儲存文件內容 ──→ [PostgreSQL（:5432）]
     │
     └── 儲存附件檔案 ──→ [MinIO S3 API（:9000）]
                              │
                              ▼
                         [/data/minio 磁碟卷]
```

### 4.3 監控資料流

```
[Uptime Kuma]
     │
     ├── HTTP 探針 ──→ [各服務 Health Check Endpoint]
     │
     ├── 告警觸發 ──→ [SMTP Server（:25）] ──→ [使用者 Email]
     │
     └── 狀態儲存 ──→ [SQLite（容器內部）]
```

## 5. 信任邊界 (Trust Boundaries)

```
╔═══════════════════════════════════════════════════════════════════════╗
║                         信任邊界示意圖                                 ║
╠═══════════════════════════════════════════════════════════════════════╣
║                                                                       ║
║  ┌─────────────────────────────────────────────────────────────────┐ ║
║  │  信任區域 1：使用者端（低信任）                                      │ ║
║  │                                                                  │ ║
║  │  • 所有流量必須經過 TLS 加密（HTTPS）                              │ ║
║  │  • 必須完成 AD/LDAP 身份驗證才能存取服務                           │ ║
║  │  • 不允許直接存取資料庫或後端服務埠                                 │ ║
║  └──────────────────────────────┬────────────────────────────────── ┘ ║
║                                 │ TLS 443（Nginx 終止加密）             ║
║   ═══════════════ 信任邊界 A ════╪════════════════════════════════════ ║
║                                 │                                      ║
║  ┌──────────────────────────────▼────────────────────────────────── ┐ ║
║  │  信任區域 2：應用服務層（中信任）                                     │ ║
║  │                                                                  │ ║
║  │  • Nginx、Outline、Homepage、Uptime Kuma、Plane                  │ ║
║  │  • 服務間透過 Docker 內部網路通訊（加密可選）                       │ ║
║  │  • 每個服務以最小權限容器使用者執行（非 root）                      │ ║
║  └──────────────────────────────┬────────────────────────────────── ┘ ║
║                                 │ Docker 內網（僅容器可路由）            ║
║   ═══════════════ 信任邊界 B ════╪════════════════════════════════════ ║
║                                 │                                      ║
║  ┌──────────────────────────────▼────────────────────────────────── ┐ ║
║  │  信任區域 3：資料層（高信任）                                        │ ║
║  │                                                                  │ ║
║  │  • PostgreSQL、Redis、MinIO                                      │ ║
║  │  • 僅 Docker 內部網路可存取，不對外暴露端口                        │ ║
║  │  • 強制密碼驗證，禁止空密碼                                        │ ║
║  └──────────────────────────────────────────────────────────────── ┘ ║
║                                                                       ║
║   ═══════════════ 信任邊界 C（外部服務）══════════════════════════════ ║
║                                                                       ║
║  ┌──────────────────────────────────────────────────────────────── ┐ ║
║  │  外部基礎設施（受管理存取）                                          │ ║
║  │                                                                  │ ║
║  │  • AD/LDAP：僅允許 Service Account 唯讀連線（LDAPS:636）          │ ║
║  │  • SMTP：僅允許單向寄信（TCP:587），不接收外部郵件                  │ ║
║  │  • NTP：僅允許時間查詢（UDP:123）                                  │ ║
║  └──────────────────────────────────────────────────────────────── ┘ ║
╚═══════════════════════════════════════════════════════════════════════╝
```

## 7. 高可用性考量 (HA Considerations)

**MVP 階段（單節點）**：

- 採用 `restart: unless-stopped` 確保容器異常後自動重啟
- 每日資料備份至獨立儲存（NAS/備援磁碟）
- 建議定期執行 VM 快照

**Phase D 以後（可選擴充）**：

| 方案 | 工具 | 說明 |
|------|------|------|
| 容器編排 | Docker Swarm | 多節點，零停機部署 |
| 資料庫高可用 | PostgreSQL Streaming Replication | 主從複製 |
| 負載均衡 | Nginx upstream | 多 App 實例 |

## 8. 備份與災難回復

```
備份策略：
┌─────────────────────────────────────────────────────────────┐
│  每日 02:00 全量備份                                          │
│  ├── PostgreSQL: pg_dump → 加密壓縮 → 傳至備份目錄            │
│  ├── MinIO: mc mirror → 備份目錄                             │
│  └── Docker Volumes: tar → 加密壓縮 → 傳至備份目錄            │
│                                                             │
│  保留期限：每日備份保留 30 天                                   │
│  異地備份：每週一次傳至備援 NAS（如有）                          │
└─────────────────────────────────────────────────────────────┘

RTO（目標回復時間）：4 小時
RPO（目標回復點）：24 小時
```
