# TITAN 稽核計畫

> 版本：1.0 | 最後更新：2026-03-23 | 適用範圍：銀行 IT 現代化平台

---

## 1. 稽核範圍

### 1.1 登入與身份驗證事件

| 事件類型 | 說明 | 優先級 |
|----------|------|--------|
| 成功登入 | 記錄使用者、時間、來源 IP、使用的 IdP | P1 |
| 失敗登入 | 記錄失敗原因、連續失敗次數 | P1 |
| 登出 | 記錄主動登出與 session 逾時 | P2 |
| MFA 驗證 | 記錄 MFA 成功／失敗 | P1 |
| 密碼變更 | 記錄變更時間與操作者 | P1 |
| Token 發行與撤銷 | 記錄 API Token 生命週期 | P1 |

### 1.2 資料存取事件

| 事件類型 | 說明 | 優先級 |
|----------|------|--------|
| 文件讀取（Outline） | 記錄存取者、文件 ID、頁面路徑 | P2 |
| 文件建立／修改／刪除 | 記錄操作者、目標資源、變更摘要 | P1 |
| 資料庫查詢（敏感表） | 記錄查詢來源、執行帳號 | P1 |
| 物件儲存存取（MinIO） | 記錄 bucket、物件名稱、操作類型 | P2 |
| 資料匯出 | 記錄匯出者、匯出範圍、目的地 | P1 |
| 備份存取 | 記錄存取備份檔案的帳號與時間 | P1 |

### 1.3 設定變更事件

| 事件類型 | 說明 | 優先級 |
|----------|------|--------|
| 系統設定變更 | 記錄變更前後差異（diff） | P1 |
| RBAC 角色變更 | 記錄角色指派、撤銷 | P1 |
| 網路設定變更 | 記錄防火牆規則、nginx 設定異動 | P1 |
| Docker Compose 變更 | 記錄服務設定異動 | P2 |
| 憑證更新 | 記錄 TLS 憑證更換時間與操作者 | P1 |
| 環境變數變更 | 記錄 .env 檔案異動（不記錄值，僅記錄鍵名） | P1 |

### 1.4 管理員操作事件

| 事件類型 | 說明 | 優先級 |
|----------|------|--------|
| 帳號建立／停用／刪除 | 記錄操作者與目標帳號 | P1 |
| 權限提升 | 記錄誰授予、誰獲得何種權限 | P1 |
| 系統維護作業 | 記錄維護視窗開始／結束、執行項目 | P2 |
| 緊急存取（break-glass） | 記錄緊急帳號啟用、使用範圍、關閉時間 | P1 |
| 安全掃描執行 | 記錄掃描時間、工具、結果摘要 | P2 |

## 2. 稽核日誌格式

### 2.1 標準日誌欄位（JSON 格式）

```json
{
  "timestamp": "2026-03-23T10:00:00+08:00",
  "event_id": "uuid-v4",
  "event_type": "AUTH_LOGIN_SUCCESS",
  "severity": "INFO",
  "actor": {
    "user_id": "user@example.com",
    "role": "Engineer",
    "source_ip": "192.168.1.100",
    "user_agent": "Mozilla/5.0 ..."
  },
  "resource": {
    "service": "outline",
    "resource_type": "document",
    "resource_id": "doc-12345",
    "action": "READ"
  },
  "outcome": "SUCCESS",
  "session_id": "sess-abcdef",
  "request_id": "req-xyz",
  "metadata": {}
}
```

### 2.2 事件嚴重級別定義

| 等級 | 說明 | 範例 |
|------|------|------|
| CRITICAL | 需立即回應的安全事件 | 權限提升攻擊、資料外洩跡象 |
| HIGH | 高風險行為，需當日處理 | 連續登入失敗、非授權存取嘗試 |
| MEDIUM | 需關注的異常行為 | 異常時間存取、大量資料下載 |
| LOW | 一般稽核記錄 | 成功登入、正常文件存取 |
| INFO | 系統正常操作記錄 | 定期備份完成、服務啟動 |

### 2.3 日誌保留政策

| 日誌類型 | 線上保留 | 冷儲存保留 | 法規依據 |
|----------|----------|-----------|----------|
| 身份驗證日誌 | 90 天 | 7 年 | 金管會資安規範 |
| 資料存取日誌 | 90 天 | 7 年 | 個資法 |
| 設定變更日誌 | 180 天 | 永久 | 內部稽核需求 |
| 管理員操作日誌 | 180 天 | 永久 | 金管會規範 |
| 系統事件日誌 | 30 天 | 1 年 | 運維需求 |

## 3. 法規遵循要求

### 3.1 金融監督管理委員會（金管會）規範

- **金融機構辦理電腦系統資訊安全評估辦法**
  - 應留存系統存取日誌至少 6 個月
  - 管理員帳號操作應全程記錄
  - 異常存取應有自動告警機制

- **銀行業資安評估基準**
  - 特權帳號管理：所有特權操作須有雙人複核日誌
  - 定期稽核：每季執行權限審查
  - 日誌完整性：日誌不可被一般使用者修改或刪除

### 3.2 個人資料保護法（個資法）

- 個人資料存取需留存稽核軌跡
- 資料外洩事件須於 72 小時內向主管機關通報
- 定期盤點個資存取授權清單

### 3.3 ISO 27001 控制項對應

| 控制項 | 說明 | TITAN 對應措施 |
|--------|------|--------------|
| A.9.2.3 | 特權存取管理 | RBAC + 稽核日誌 |
| A.9.4.2 | 安全登入程序 | SSO + MFA + 登入稽核 |
| A.12.4.1 | 事件日誌記錄 | 集中式日誌收集 |
| A.12.4.3 | 管理員與操作員日誌 | 管理員操作全程記錄 |
| A.18.1.3 | 記錄保護 | 日誌加密儲存、存取控制 |

## 4. 稽核審查排程

### 4.1 每日自動稽核

- 時間：每日 06:00（Asia/Taipei）
- 執行方式：`scripts/audit-check.sh` 排程執行
- 項目：
  - 失敗登入嘗試統計
  - 憑證到期日檢查（< 30 天告警）
  - 備份完成狀態確認
  - 服務健康狀態

### 4.2 每週稽核日誌審查

- 時間：每週一 09:00
- 負責人：IT 安全工程師
- 審查項目：
  1. 過去 7 天登入異常事件
  2. 權限變更記錄
  3. 資料存取異常模式
  4. 自動稽核報告彙整
  5. 未處理告警清單

### 4.3 每月合規審查

- 時間：每月最後一個工作日
- 負責人：IT 經理 + 資安主管
- 審查項目：
  1. RBAC 權限矩陣對比（實際 vs. 核准清單）
  2. 帳號生命週期審查（離職員工帳號清理確認）
  3. 合規指標儀表板審查
  4. 事件回應紀錄彙整

### 4.4 每季特權帳號審查

- 時間：每季第一個工作週
- 負責人：IT 主管 + 稽核部門
- 審查項目：
  1. Admin 帳號清單複核
  2. 緊急存取（break-glass）使用紀錄
  3. 第三方供應商存取權限審查
  4. 下季度稽核計畫調整

## 5. 事件回應流程

### 5.1 安全事件分級回應時限

| 事件等級 | 初始回應 | 調查完成 | 通報時限 |
|----------|----------|---------|---------|
| CRITICAL | 15 分鐘 | 4 小時 | 立即通報主管 |
| HIGH | 1 小時 | 24 小時 | 當日通報 |
| MEDIUM | 4 小時 | 72 小時 | 週報彙整 |
| LOW | 24 小時 | 7 天 | 月報彙整 |

### 5.2 聯絡清單

| 角色 | 職責 |
|------|------|
| IT 安全工程師 | 日常稽核監控、初始事件調查 |
| IT 經理 | HIGH 以上事件授權處理、月報審核 |
| 資安主管 | CRITICAL 事件決策、對外通報 |
| 法遵部門 | 監理報告、個資事件通報 |
