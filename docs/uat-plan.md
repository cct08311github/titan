# TITAN — 使用者驗收測試計畫（UAT）

> T23 | 對應 Issue #34

---

## 1. UAT 目標與範圍

### 1.1 目標

由實際使用者（業務代表）在接近生產的環境中，驗證 TITAN 平台功能符合業務需求，並確認使用者體驗符合預期。

### 1.2 UAT 範圍

| 功能模組 | 驗收重點 |
|----------|----------|
| 使用者認證 | 登入、登出、密碼管理 |
| Plane 任務管理 | 建立/編輯/刪除任務、看板操作、Sprint 管理 |
| Outline 文件協作 | 文件建立/編輯、集合管理、搜尋 |
| 通知與整合 | 任務指派通知、Email 通知 |
| 權限控制 | 角色權限正確隔離 |

### 1.3 不在 UAT 範圍

- 系統效能壓力測試
- 第三方整合（Phase F）
- 行動裝置測試（Phase F）

## 4. UAT 場景

### UAT-01：每日工作起始流程

**業務情境：** 員工每日上班第一件事，確認今日待辦任務並查閱相關文件。

**步驟：**
1. 登入 TITAN（`uat.titan.internal`）
2. 在 Plane 看板上找到指派給自己的任務
3. 點開任務，閱讀描述與附件
4. 點擊任務中的文件連結，開啟 Outline 相關規格
5. 確認自己的工作優先順序

**驗收標準：**
- [ ] 登入在 5 秒內完成
- [ ] 看板清楚顯示個人任務
- [ ] 文件連結可正常開啟
- [ ] 整體操作流暢，無需技術支援

**使用者評分（1-5 分）：** ___

---

### UAT-02：建立與管理任務

**業務情境：** PM 建立新需求任務，並指派給開發人員。

**步驟：**
1. 在 Plane 建立新任務
2. 填寫：標題、描述（含 Markdown 格式）、優先級、截止日期
3. 指派給另一位測試成員
4. 設定標籤（Label）
5. 移動任務至「In Progress」狀態
6. 確認被指派人看到任務更新

**驗收標準：**
- [ ] 任務建立操作直覺，無需查閱手冊
- [ ] Markdown 格式在描述中正確渲染
- [ ] 指派動作即時通知被指派人
- [ ] 狀態拖拽操作流暢

**使用者評分（1-5 分）：** ___

---

### UAT-03：文件協作編輯

**業務情境：** 開發人員在 Outline 撰寫技術文件，並與同事協作。

**步驟：**
1. 在 Outline 建立新集合：「UAT 測試文件」
2. 新增文件並撰寫內容（含標題、段落、表格、程式碼區塊）
3. 分享文件連結給另一位測試成員
4. 另一位成員開啟文件並新增留言
5. 原作者查看留言並回覆
6. 確認文件版本歷史記錄

**驗收標準：**
- [ ] 文件編輯器操作直覺（類似 Notion）
- [ ] 文件格式渲染正確
- [ ] 分享連結可正常存取
- [ ] 留言功能正常運作
- [ ] 版本歷史可查閱

**使用者評分（1-5 分）：** ___

---

### UAT-04：全文搜尋

**業務情境：** 使用者需要找到過去建立的文件或任務。

**步驟：**
1. 在搜尋框輸入關鍵字（文件標題的一部分）
2. 確認搜尋結果包含相關文件
3. 輸入任務關鍵字，確認 Plane 任務出現在結果中
4. 使用進階篩選（日期範圍、建立者）
5. 確認搜尋速度符合期望

**驗收標準：**
- [ ] 搜尋回應在 2 秒內
- [ ] 結果排序合理（最相關優先）
- [ ] 進階篩選功能可用

**使用者評分（1-5 分）：** ___

---

### UAT-05：通知與提醒

**業務情境：** 確認系統通知機制正常運作。

**步驟：**
1. 以使用者 A 指派任務給使用者 B
2. 確認使用者 B 收到系統通知（App 內 + Email）
3. 以使用者 A 在使用者 B 的任務留言 @mention
4. 確認使用者 B 收到 @mention 通知
5. 任務截止日前 24 小時，確認提醒通知

**驗收標準：**
- [ ] 指派通知在 5 分鐘內送達
- [ ] Email 通知格式清楚（含任務連結）
- [ ] @mention 通知正確觸發

**使用者評分（1-5 分）：** ___

---

### UAT-06：權限控制驗證

**業務情境：** 確認不同角色的存取權限正確隔離。

**步驟：**
1. 以「唯讀成員」角色登入
2. 嘗試建立新任務（應被拒絕）
3. 嘗試刪除他人文件（應被拒絕）
4. 確認可以查看但無法修改
5. 以「管理員」角色確認可執行上述操作

**驗收標準：**
- [ ] 唯讀角色無法建立/修改/刪除
- [ ] 權限拒絕時顯示清楚提示訊息（非白畫面/500 錯誤）
- [ ] 管理員角色權限完整

**使用者評分（1-5 分）：** ___

## 5. 驗收標準（全域）

| 項目 | 標準 |
|------|------|
| 場景通過率 | ≥ 90%（6 個場景中至少 5 個完全通過） |
| 平均使用者評分 | ≥ 4.0 / 5.0 |
| 重大缺陷 | 0 個 P0/P1 未解決缺陷 |
| 使用者整體滿意度 | ≥ 80% 使用者評「符合或超過預期」 |

## 7. UAT 執行時程

| 天次 | 活動 | 負責人 |
|------|------|--------|
| Day 1 上午 | UAT 開幕說明、環境驗證、帳號發放 | PM + QA |
| Day 1 下午 | UAT-01、UAT-02 執行 | 業務代表 |
| Day 2 上午 | UAT-03、UAT-04 執行 | 業務代表 |
| Day 2 下午 | UAT-05、UAT-06 執行 | 業務代表 |
| Day 3 | 缺陷修復（P1 以上）+ 回歸驗證 | Dev + QA |
| Day 4 | 重新執行失敗場景 | 業務代表 |
| Day 5 | UAT 結果確認、回饋收集、報告 | PM + QA |
