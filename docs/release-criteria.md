# TITAN 上線標準 — Go/No-Go 決策清單

> 版本: v1.0 | 最後更新: 2026-03-25
>
> 本文件定義 TITAN v1.0 正式上線前的 Go/No-Go 決策標準。
> 所有「必要」項目必須通過才能做出 Go 決策。

---

## 決策機制 (Go/No-Go Decision Rules)

| 條件 | 決策 |
|------|------|
| 所有「必要」項目通過 | **Go** — 可上線 |
| 有「必要」項目未通過 | **No-Go** — 修復後重新評估 |
| 僅「建議」項目未通過 | **有條件 Go** — 上線但限期改善 |

**決策會議出席者**: PM、Tech Lead、QA Lead、Infosec、Ops

## 3. 安全 (Security)

| # | 標準 | 驗證方法 | 等級 | 狀態 |
|---|------|---------|------|------|
| S-01 | 無 Critical/High 漏洞 | `npm audit` + `scripts/security-scan.sh` | 必要 | - [ ] |
| S-02 | CSP header 正確設定 | 檢查 response header，確認 nonce 機制 | 必要 | - [ ] |
| S-03 | JWT 簽名驗證正確 | Edge middleware 測試 + 手動測試 | 必要 | - [ ] |
| S-04 | Rate limiting 生效 | 短時間內大量 request，確認被限流 | 必要 | - [ ] |
| S-05 | RBAC 權限控制正確 | 各角色嘗試越權操作，確認被拒絕 | 必要 | - [ ] |
| S-06 | 密碼策略符合規範 | 測試弱密碼被拒絕 | 必要 | - [ ] |
| S-07 | 無敏感資訊外洩 | 檢查 API response 不含密碼 hash、內部路徑 | 必要 | - [ ] |
| S-08 | HTTPS 強制啟用 | 確認 HTTP → HTTPS 重導向 | 必要 | - [ ] |
