# VM 資源需求申請 (VM Resource Request)

> **任務**: T05 — VM 資源申請
> **Issue**: #6
> **版本**: v1.0
> **日期**: 2026 年 3 月 23 日

---

## 1. 需求摘要

TITAN 平台採用 Docker Compose 單節點部署架構，依環境性質規劃三套 VM 規格。

| 環境 | 主機名稱 | 用途 | 優先順序 |
|------|---------|------|---------|
| Production（生產） | `titan-prod-01` | 正式服務，5 位使用者每日使用 | 最高 |
| UAT（驗收測試） | `titan-uat-01` | 驗收測試、升級驗證 | 高 |
| Dev（開發）| `titan-dev-01` | 功能開發、整合測試 | 中 |

> **MVP 階段最低需求**：僅 Production + UAT 兩台即可啟動。Dev 環境為選配，可延後申請。

---

## 2. VM 規格矩陣

### 2.1 Production 環境

| 項目 | 最低規格 | 建議規格 | 說明 |
|------|---------|---------|------|
| **vCPU** | 4 核心 | 8 核心 | 多服務同時執行，建議 8 核以應對尖峰 |
| **RAM** | 8 GB | 16 GB | PostgreSQL 2G + Outline 1G + 各服務緩衝 |
| **OS 磁碟** | 50 GB | 80 GB | OS + Docker Engine + 映像儲存 |
| **資料磁碟** | 200 GB | 500 GB | PostgreSQL 資料 + MinIO 附件 + Volumes |
| **網路介面** | 1 Gbps | 1 Gbps | 標準內網速度 |
| **OS** | Rocky Linux 9.x | Rocky Linux 9.x | 或 Ubuntu Server 22.04 LTS |

**磁碟分配詳情（建議規格）**：

```
OS 磁碟（80 GB）：
  /boot          1 GB
  /              30 GB（OS + 應用程式）
  /var/lib/docker 49 GB（Docker 映像、容器層）

資料磁碟（500 GB）：
  /data/titan/postgres   100 GB
  /data/titan/minio      300 GB
  /data/titan/redis        5 GB
  /data/titan/outline     50 GB
  /data/titan/uptime-kuma 10 GB
  /data/backup            35 GB（本機備份暫存）
```

## 5. 資源計算說明

### 5.1 記憶體需求分析（Production）

| 服務 | 記憶體配置上限 | 說明 |
|------|-------------|------|
| postgres | 2,048 MB | `docker-compose.yml` 設定值 |
| redis | 512 MB | `docker-compose.yml` 設定值 |
| minio | 1,024 MB | S3 物件儲存 |
| outline | 1,024 MB | 知識庫 Web 服務 |
| homepage | 256 MB | 靜態入口頁，低需求 |
| uptime-kuma | 512 MB | 監控服務 |
| nginx | 256 MB | 反向代理，低需求 |
| **容器總計** | **5,632 MB** | 約 5.5 GB |
| OS + Docker Daemon | 2,000 MB | 作業系統基本需求 |
| 緩衝空間 | 1,000 MB | 避免 OOM |
| **建議 RAM** | **≥ 8,632 MB** | **取整為 16 GB（含成長空間）** |

### 5.2 CPU 需求分析

| 服務 | CPU 配置上限 | 說明 |
|------|------------|------|
| postgres | 2.0 核心 | `docker-compose.yml` 設定值 |
| redis | 1.0 核心 | |
| minio | 1.0 核心 | |
| outline | 1.0 核心 | |
| homepage | 0.5 核心 | |
| uptime-kuma | 0.5 核心 | |
| nginx | 0.5 核心 | |
| **容器總計** | **6.5 核心** | |
| **建議 vCPU** | **8 核心** | 含 OS 及並發緩衝 |

### 5.3 磁碟需求分析

| 類型 | 容量 | 成長估計（1年）| 建議配置 |
|------|------|-------------|---------|
| Docker 映像 | ~1 GB | 版本更新增量小 | 10 GB |
| PostgreSQL 資料 | 初始 < 1 GB | 5 人團隊 50 GB/年 | 100 GB |
| MinIO 附件 | 初始 0 | 文件附件 100 GB/年 | 200 GB |
| Redis 資料 | < 1 GB | 穩定 | 5 GB |
| 日誌與備份暫存 | 10 GB | 依保留期限 | 50 GB |
| **資料磁碟總計** | | | **≥ 365 GB → 取整 500 GB** |
