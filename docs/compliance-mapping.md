# TITAN 合規映射表

**文件版本**: v1.0
**撰寫日期**: 2026-03-24
**適用對象**: 資訊長（CIO）、資安主管、稽核人員
**機密等級**: 內部限閱
**審核週期**: 每年複審，法規修訂時立即更新

---

## 1. 概述

本文件說明 TITAN 工作管理系統如何對應銀行資安法規要求及 ISO 27001 資訊安全管理標準，並列舉系統內建的安全控制機制與稽核留底功能，供資安稽核參考使用。

## 3. TITAN 安全控制機制對照

### 3.1 存取控制（Access Control）

| 安全控制 | 技術實作 | 對應法規/標準 |
|---------|---------|-------------|
| **JWT 認證** | 使用 JWT（JSON Web Token）進行 API 認證，Token 有效期限 15 分鐘（Access Token）+ 7 天（Refresh Token） | ISO 27001 A.9.4.2 安全的登入程序 |
| **JWT Blacklist（Token 撤銷）** | Redis 維護已撤銷 Token 黑名單，登出後 Token 立即失效，防止 Token 竊取後被濫用 | ISO 27001 A.9.4.2；金管會資安守則 §存取控制 |
| **Session Timeout（閒置超時）** | 使用者閒置 30 分鐘後自動登出，Session 資料於伺服器端清除 | ISO 27001 A.11.2.8 無人看管的使用者設備；金管會資安評估辦法 |
| **帳號鎖定機制** | 連續登入失敗 5 次後帳號鎖定 15 分鐘（可設定），防止暴力破解 | ISO 27001 A.9.4.2；金管會資安守則 §密碼管理 |
| **RBAC 角色權限控制** | 使用者依角色（管理員/一般使用者）分配功能存取權限，最小權限原則 | ISO 27001 A.9.1.2 資訊存取限制；ISO 27002 A.8.2 |

---

### 3.2 應用程式安全（Application Security）

| 安全控制 | 技術實作 | 對應法規/標準 |
|---------|---------|-------------|
| **CSRF 防護** | 所有狀態變更操作（POST/PUT/PATCH/DELETE）均驗證 CSRF Token，防止跨站請求偽造攻擊 | OWASP Top 10 A01；ISO 27001 A.14.1.2 |
| **Rate Limiting（速率限制）** | API 層實作速率限制（每 IP 每分鐘 60 次 API 請求，登入端點每 IP 每分鐘 5 次），防止 DoS 及暴力破解 | ISO 27001 A.12.1.3 容量管理；金管會資安守則 §網路安全 |
| **Input Validation（輸入驗證）** | 所有 API 端點使用 Zod schema 驗證輸入，防止 Injection 攻擊 | OWASP Top 10 A03；ISO 27001 A.14.2.5 |
| **SQL Injection 防護** | 全面使用 Prisma ORM 參數化查詢，不拼接原始 SQL | OWASP Top 10 A03；金管會資安評估辦法 §應用安全 |
| **XSS 防護** | Next.js 框架內建 React 的自動 HTML 跳脫；HTTP 安全標頭（Content-Security-Policy）| OWASP Top 10 A07；ISO 27001 A.14.1.2 |
| **安全 HTTP 標頭** | 設定 X-Frame-Options、X-Content-Type-Options、HSTS、Referrer-Policy 等安全標頭 | OWASP Security Headers；ISO 27001 A.14.1.2 |

---

### 3.3 稽核與記錄（Audit & Logging）

| 安全控制 | 技術實作 | 對應法規/標準 |
|---------|---------|-------------|
| **Audit Log（稽核日誌）** | 所有使用者操作（建立、修改、刪除任務；使用者管理；系統設定變更）均記錄操作時間、操作人、操作類型、變更前後值 | ISO 27001 A.12.4.1 事件記錄；金管會資安評估辦法 §稽核追蹤 |
| **登入記錄** | 記錄每次登入成功/失敗，包含時間戳、來源 IP、使用者 ID，保存 90 天 | ISO 27001 A.9.4.2；金管會資安守則 §存取紀錄 |
| **API 存取日誌** | Nginx/Next.js 記錄所有 API 請求（端點、方法、回應碼、回應時間），保存 30 天 | ISO 27001 A.12.4.1 |
| **管理員操作記錄** | 帳號建立/停用/解鎖、角色變更等管理操作均有獨立稽核記錄 | ISO 27001 A.9.2 使用者存取管理 |
| **日誌保護** | 稽核日誌寫入後不可修改（僅追加寫入），日誌檔案權限限制為唯讀 | ISO 27001 A.12.4.2 日誌資訊保護 |

---

### 3.4 資料保護（Data Protection）

| 安全控制 | 技術實作 | 對應法規/標準 |
|---------|---------|-------------|
| **密碼雜湊** | 使用者密碼以 bcrypt（cost factor 12）雜湊儲存，不儲存明文密碼 | ISO 27001 A.9.4.3；金管會資安守則 §密碼管理 |
| **傳輸加密** | 內網部署使用 TLS/HTTPS（自簽憑證或內網 CA），確保資料傳輸加密 | ISO 27001 A.10.1 密碼控制；金管會資安守則 §傳輸安全 |
| **敏感資料遮罩** | API 回應不返回密碼雜湊、內部 Token 等敏感欄位 | ISO 27001 A.14.1.2 |
| **資料最小化** | 系統僅收集業務必要的個人資料，不儲存超出需求的使用者資訊 | 個人資料保護法 §11；ISO 27001 A.8.2 |
| **備份加密** | 資料庫備份檔案於儲存時進行加密，防止備份媒體遺失造成資料洩漏 | ISO 27001 A.12.3 備份 |

---

### 3.5 網路與基礎設施安全（Infrastructure Security）

| 安全控制 | 技術實作 | 對應法規/標準 |
|---------|---------|-------------|
| **內網隔離部署** | TITAN 僅部署於封閉內部網路，不對外網開放，透過 Nginx 反向代理統一入口 | ISO 27001 A.13.1.1 網路控制；金管會資安守則 §網路架構 |
| **容器化隔離** | 使用 Docker 容器化各服務（應用、資料庫、快取），服務間通過內部網路通訊 | ISO 27001 A.13.1.3 網路隔離 |
| **最小化開放端口** | 僅開放必要的服務端口（443/80 for Web），資料庫端口（5432）不對外開放 | ISO 27001 A.13.1.1 |
| **定期安全更新** | 系統依賴套件（npm packages、Docker base images）每月定期更新，CVE 高危漏洞 P0 立即修補 | ISO 27001 A.12.6.1 技術弱點管理 |

## 5. 稽核留底功能對照

### 5.1 稽核查詢能力

TITAN 的 Audit Log 模組提供以下查詢能力，可直接支援稽核作業：

| 稽核需求 | TITAN 提供的查詢 |
|---------|----------------|
| 特定期間內誰存取了系統 | 依時間區間查詢登入記錄 |
| 誰修改了哪個任務 | 依任務 ID 查詢變更歷史（含修改前後值）|
| 哪些帳號被建立/停用 | 使用者管理操作稽核日誌 |
| 是否有異常登入嘗試 | 登入失敗記錄（含來源 IP）|
| 管理員執行了哪些特權操作 | 管理員操作稽核日誌 |
| 資料最後一次修改時間 | 所有記錄有 `updated_at` 時間戳及操作人 |

### 5.2 日誌保存期限

| 日誌類型 | 保存期限 | 儲存位置 |
|---------|---------|---------|
| 稽核日誌（Audit Log）| 1 年（滾動保存）| PostgreSQL（可匯出 CSV）|
| 登入/登出記錄 | 90 天 | PostgreSQL |
| API 存取日誌 | 30 天 | 伺服器日誌檔案 |
| 錯誤日誌 | 30 天 | 伺服器日誌檔案 |
| 備份操作記錄 | 永久 | 備份操作日誌 |

> 注意：若銀行資安政策要求更長保存期限，可調整資料庫配置或設定日誌歸檔。

## 7. 資料保護措施摘要

### 7.1 個人資料保護

TITAN 系統儲存的個人資料範圍有限：

| 資料項目 | 儲存目的 | 保護措施 |
|---------|---------|---------|
| 員工姓名 | 任務指派、稽核追蹤 | 僅授權使用者可查詢 |
| 員工帳號（工號/Email）| 身分認證 | 登入使用，不對外顯示 |
| 密碼（雜湊值）| 身分驗證 | bcrypt 單向雜湊，不可還原 |
| 操作 IP 位址 | 安全稽核 | 僅管理員可查詢，90 天後清除 |

> TITAN 不儲存員工薪資、身分證號碼、個人健康資訊等高敏感個人資料。
