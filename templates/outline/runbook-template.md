## 初步評估（5 分鐘內完成）

### 1. 確認事故範圍

- [ ] 哪些服務受影響？
- [ ] 影響使用者數量？
- [ ] 是否影響核心交易？

### 2. 收集初步資訊

```bash

docker compose ps

# 查看最近錯誤日誌

```

### 3. 初步嚴重度判斷

| P1 | 核心服務中斷，無法交易 |
|----|----------------------|
| P2 | 部分功能異常，有替代方案 |
| P3 | 效能下降，不影響主要功能 |
| P4 | 輕微問題，可計畫性處理 |
