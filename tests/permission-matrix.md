# TITAN 權限矩陣測試計畫

> 版本：1.0 | 更新：2026-03-23
> 用途：UAT（使用者驗收測試）期間驗證各角色存取權限是否符合設計規範

---

## 1. 角色定義

| 角色代碼 | 角色名稱 | 說明 |
|---------|---------|------|
| ADMIN | 系統管理員 | 最高權限，可管理所有資源與使用者 |
| MANAGER | IT 經理 | 可查看所有資源、審核報告、管理下屬 |
| ENGINEER | IT 工程師 | 可操作日常 IT 服務，有限制的設定存取 |
| READONLY | 唯讀人員 | 僅可查閱文件，不可進行任何修改 |

## 4. 權限邊界測試（負面測試）

下列測試案例專門測試「不應被允許的操作」，確認系統有正確拒絕。

| 測試 ID | 角色 | 嘗試的操作 | 預期系統回應 | 結果 |
|---------|------|-----------|------------|------|
| NEG-001 | ENGINEER | 嘗試刪除他人建立的文件 | HTTP 403 / 操作被拒絕 | ☐ |
| NEG-002 | MANAGER | 嘗試存取 MinIO Admin API | HTTP 403 | ☐ |
| NEG-003 | READONLY | 嘗試透過 API 建立 Outline 文件 | HTTP 403 | ☐ |
| NEG-004 | ENGINEER | 嘗試提升自己的角色 | 操作被拒絕 | ☐ |
| NEG-005 | 任何角色 | 嘗試存取其他使用者的 Session Token | 無法取得或使用 | ☐ |
| NEG-006 | READONLY | 嘗試透過直接 URL 存取設定頁面 | 重導向至登入頁或顯示 403 | ☐ |

## 5. UAT 執行說明

### 5.1 執行前準備

```bash
# 確認各角色測試帳號已建立
# ADMIN 帳號：admin@titan.internal
# MANAGER 帳號：manager@titan.internal
# ENGINEER 帳號：engineer@titan.internal
# READONLY 帳號：readonly@titan.internal

# 確認測試環境與生產環境隔離
# 確認測試結束後清除測試資料
```

### 5.2 測試記錄方式

- 每個測試案例完成後在「結果」欄位標記：
  - `☑ PASS` — 符合預期
  - `✗ FAIL` — 不符合預期（需記錄於備註欄並建立 Bug Issue）
  - `⊘ SKIP` — 因環境問題跳過（需說明原因）

### 5.3 完成標準

- 所有 `P1` 測試案例必須 PASS 才可上線
- FAIL 項目需修復並重新測試
- 測試結果需由 IT 經理簽核
