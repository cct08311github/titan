# 1. 管理摘要 (Executive Summary)

## 1.1 問題陳述

本行資訊工程團隊（5 人編制）目前面臨嚴峻的工作效率瓶頸。日常作業高度仰賴 **Email 傳遞**、**Excel 追蹤**、**紙本文件**及**各自獨立的內部系統**，導致資訊散落於多個不連通的管道中。團隊成員每週浪費大量時間在「找資料」而非「做事情」，更面臨版本衝突、資訊遺失、缺乏稽核軌跡等系統性風險。

## 1.2 解決方案概述

我們提議採用 **「開源核心 + 自製膠合層」** 的混合架構策略：

```
┌─────────────────────────────────────────────────────┐
│                    解決方案核心理念                      │
├─────────────────────────────────────────────────────┤
│  ✦ 7 套開源工具 → 覆蓋任務、知識、程式碼、監控全生命週期   │
│  ✦ n8n 自動化平台 → 串接所有工具，打造無縫工作流          │
│  ✦ Docker 容器化 → 外網開發、內網部署，完全離線運行        │
│  ✦ 統一入口 Homepage → 一個頁面進入所有系統               │
└─────────────────────────────────────────────────────┘
```

**開發模式**：在外部網路環境搭配 AI 輔助開發客製化整合層 → 打包為 Docker 映像 → 部署至銀行封閉內網。

## 1.3 預期效益

| 指標 | 預期改善 |
|------|---------|
| 資訊查找時間 | 減少 **70%** |
| 任務追蹤效率 | 提升 **80%** |
| 變更管理合規性 | 從 **無** 到 **100% 可追溯** |
| 知識傳承風險 | 從 **高** 降至 **低** |
| 系統可用性掌握 | 從 **被動發現** 到 **主動告警** |

## 1.4 時程與成本

| 項目 | 數值 |
|------|------|
| 總建置時程 | **3 個月**（分 4 階段） |
| 軟體授權費 | **$0**（全部開源） |
| 硬體成本 | **$0**（利用現有伺服器） |
| 唯一成本 | 團隊工程時間（約 **240 人時**） |

## 2.2 具體痛點分析

### 痛點一：Email 作為溝通與任務追蹤工具

| 問題 | 影響 |
|------|------|
| 信件淹沒在收件匣 | 重要任務遺漏，無法有效排序優先級 |
| 多人回覆造成版本分歧 | 「請以最新版為主」成為常見但無效的口頭禪 |
| 離職/調職時信件不可轉移 | 歷史脈絡永久流失 |
| 無法統計工作負載 | 主管難以評估團隊能量與瓶頸 |

### 痛點二：Excel 作為專案管理工具

| 問題 | 影響 |
|------|------|
| 檔案鎖定衝突 | 多人無法同時編輯，效率低下 |
| 版本混亂（v1, v2, final, final_final） | 永遠不確定哪個是最新版 |
| 無法設定自動提醒 | 逾期任務無人追蹤 |
| 格式因人而異 | 彙整報表時需大量人工對齊 |

### 痛點三：紙本文件與簽核流程

| 問題 | 影響 |
|------|------|
| 簽核流程耗時 | 平均一份 CR 簽核需 3-5 個工作天 |
| 無法全文搜尋 | 找一份半年前的文件可能需要數小時 |
| 實體保管風險 | 紙本遺失即永久遺失 |
| 無法遠端作業 | 疫情等突發狀況時流程完全中斷 |

### 痛點四：程式碼與變更管理

| 問題 | 影響 |
|------|------|
| 無版本控制系統 | 無法追溯「誰在什麼時候改了什麼」 |
| CR (Change Request) 以紙本為主 | 稽核時難以對應程式碼變更 |
| 缺乏 Code Review 機制 | 品質把關僅靠個人自律 |
| 部署流程不標準化 | 每次部署都是「手動冒險」 |

## 2.3 時間浪費估算

以 5 人團隊、每週 40 小時工作時間計算：

| 浪費項目 | 每人每週 | 團隊每週 | 團隊每月 | 團隊每年 |
|---------|:-------:|:-------:|:-------:|:-------:|
| 搜尋散落的 Email/文件 | 3.0 hr | 15.0 hr | 60 hr | **720 hr** |
| 處理 Excel 版本衝突 | 1.5 hr | 7.5 hr | 30 hr | **360 hr** |
| 紙本簽核等待與跑流程 | 2.0 hr | 10.0 hr | 40 hr | **480 hr** |
| 重複回答相同問題（無知識庫） | 1.5 hr | 7.5 hr | 30 hr | **360 hr** |
| 手動檢查系統狀態 | 1.0 hr | 5.0 hr | 20 hr | **240 hr** |
| **合計** | **9.0 hr** | **45.0 hr** | **180 hr** | **2,160 hr** |

> **結論**：團隊每年浪費約 **2,160 小時**（相當於 **1.04 個全職人力**）在可自動化或可優化的流程上。
> 以平均人力成本計算，這相當於每年 **超過 100 萬元** 的隱性損失。

## 3.3 工具選型與說明

| 工具 | 用途 | 取代對象 | 選型理由 |
|------|------|---------|---------|
| **[Plane](https://plane.so)** | 專案與任務管理 | Excel 追蹤表、Email 指派 | 開源、功能媲美 Jira、Docker 原生支援、具備看板/甘特圖/衝刺管理 |

| **[Grafana](https://grafana.com)** | 指標視覺化儀表板 | 無（或 Excel 報表） | 開源、業界標準、強大圖表引擎、支援多種資料源 |

## 4.2 Phase 1：基礎協作平台（第 1-2 週）

**目標**：建立團隊協作的基礎設施，讓日常任務管理與知識管理立即上線。

| 工作項目 | 負責人 | 天數 | 產出 |
|---------|--------|:----:|------|
| Docker 環境建置 | 系統工程師 | 1 | Docker Engine + Compose 運行環境 |
| PostgreSQL + Redis 部署 | 系統工程師 | 0.5 | 共用資料庫與快取服務 |
| Plane 部署與設定 | 專案負責人 | 1 | 任務管理系統上線 |
| Outline 部署與設定 | 專案負責人 | 1 | 知識庫系統上線 |
| Homepage 部署與設定 | 前端工程師 | 0.5 | 統一入口頁面 |
| LDAP/AD 整合 | 系統工程師 | 2 | 統一帳號認證 |
| 資料遷移（Excel → Plane） | 全體 | 2 | 現有任務全部匯入 |
| 團隊教育訓練 | 專案負責人 | 1 | 全員熟悉基本操作 |

**Phase 1 驗收標準**：
- [ ] 全體成員可透過 AD 帳號登入 Plane 與 Outline
- [ ] 現有待辦任務已從 Excel 遷移至 Plane

- [ ] 至少 3 份關鍵文件已遷移至 Outline

## 4.3 Phase 2：版本控制與變更管理（第 3-4 週）

# 5. 部署配置 (Docker Compose Configuration)

## 5.1 Phase 1 完整 Docker Compose 設定

以下為 Phase 1 的完整 `docker-compose.yml`，涵蓋 Plane、Outline、Homepage 及其相依服務。

```yaml
# =============================================================================
# 銀行 IT 現代化平台 - Phase 1 Docker Compose
# 包含: Plane (任務管理) + Outline (知識庫) + Homepage (入口) + 基礎服務
# =============================================================================
# 使用方式:
#   1. 在外網環境: docker compose build && docker compose up -d
#   2. 匯出映像:   docker save $(docker compose images -q) -o phase1-images.tar
#   3. 內網載入:   docker load -i phase1-images.tar
#   4. 內網啟動:   docker compose up -d
# =============================================================================

version: "3.9"

# ---------------------------------------------------------------------------
# 共用網路 - 所有服務透過內部網路互通
# ---------------------------------------------------------------------------
networks:
  bank-internal:
    driver: bridge
    ipam:
      config:
        - subnet: 172.20.0.0/16

# ---------------------------------------------------------------------------
# 持久化儲存 - 確保資料不因容器重啟而遺失
# ---------------------------------------------------------------------------
volumes:
  postgres-data:
    driver: local
  redis-data:
    driver: local
  minio-data:
    driver: local
  outline-data:
    driver: local
  plane-data:
    driver: local

services:

  # =========================================================================
  # 基礎服務層 (Infrastructure)
  # =========================================================================

  # -------------------------------------------------------------------------
  # PostgreSQL - 主要關聯式資料庫
  # 用途: Plane 與 Outline 共用的資料庫後端
  # -------------------------------------------------------------------------
  postgres:
    image: postgres:16-alpine
    container_name: bank-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: bankadmin
      POSTGRES_PASSWORD: "${POSTGRES_PASSWORD}"          # 請設定強密碼
      POSTGRES_MULTIPLE_DATABASES: plane,outline          # 啟動時自動建立
    volumes:
      - postgres-data:/var/lib/postgresql/data
      # 掛載初始化腳本，自動建立多個資料庫
      - ./init-scripts/create-databases.sh:/docker-entrypoint-initdb.d/create-databases.sh
    ports:
      - "5432:5432"
    networks:
      - bank-internal
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U bankadmin"]
      interval: 10s
      timeout: 5s
      retries: 5

  # -------------------------------------------------------------------------
  # Redis - 快取與訊息佇列
  # 用途: Plane 的即時功能、Outline 的快取、n8n 的佇列
  # -------------------------------------------------------------------------
  redis:
    image: redis:7-alpine
    container_name: bank-redis
    restart: unless-stopped
    command: redis-server --requirepass "${REDIS_PASSWORD}"
    volumes:
      - redis-data:/data
    ports:
      - "6379:6379"
    networks:
      - bank-internal
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # -------------------------------------------------------------------------
  # MinIO - S3 相容物件儲存
  # 用途: Outline 的檔案附件、Plane 的附件儲存
  # -------------------------------------------------------------------------
  minio:
    image: minio/minio:latest
    container_name: bank-minio
    restart: unless-stopped
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: "${MINIO_ACCESS_KEY}"
      MINIO_ROOT_PASSWORD: "${MINIO_SECRET_KEY}"
    volumes:
      - minio-data:/data
    ports:
      - "9000:9000"        # API 端口
      - "9001:9001"        # 管理介面
    networks:
      - bank-internal
    healthcheck:
      test: ["CMD", "mc", "ready", "local"]
      interval: 10s
      timeout: 5s
      retries: 5

  # =========================================================================
  # 應用服務層 (Applications)
  # =========================================================================

  # -------------------------------------------------------------------------
  # Plane - 專案與任務管理平台
  # 取代: Excel 追蹤表、Email 任務指派
  # 功能: 看板、甘特圖、衝刺管理、自訂工作流
  # 文件: https://docs.plane.so/self-hosting/docker-compose
  # -------------------------------------------------------------------------
  plane-web:
    image: makeplane/plane-frontend:stable
    container_name: bank-plane-web
    restart: unless-stopped
    depends_on:
      plane-api:
        condition: service_healthy
    environment:
      NEXT_PUBLIC_API_BASE_URL: "http://plane-api:8000"
    ports:
      - "3000:3000"
    networks:
      - bank-internal

  plane-api:
    image: makeplane/plane-backend:stable
    container_name: bank-plane-api
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      DATABASE_URL: "postgresql://bankadmin:${POSTGRES_PASSWORD}@postgres:5432/plane"
      REDIS_URL: "redis://:${REDIS_PASSWORD}@redis:6379/0"
      SECRET_KEY: "${PLANE_SECRET_KEY}"
      # 封閉網路設定 - 停用所有外部連線
      ENABLE_SIGNUP: "1"
      ENABLE_EMAIL_PASSWORD: "1"
      CORS_ALLOWED_ORIGINS: "http://portal.bank.local,http://plane.bank.local"
    ports:
      - "8000:8000"
    networks:
      - bank-internal
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/api/health/"]
      interval: 15s
      timeout: 10s
      retries: 5

  plane-worker:
    image: makeplane/plane-backend:stable
    container_name: bank-plane-worker
    restart: unless-stopped
    depends_on:
      plane-api:
        condition: service_healthy
    command: ["celery", "-A", "plane", "worker", "-l", "info"]
    environment:
      DATABASE_URL: "postgresql://bankadmin:${POSTGRES_PASSWORD}@postgres:5432/plane"
      REDIS_URL: "redis://:${REDIS_PASSWORD}@redis:6379/0"
      SECRET_KEY: "${PLANE_SECRET_KEY}"
    networks:
      - bank-internal

  plane-beat:
    image: makeplane/plane-backend:stable
    container_name: bank-plane-beat
    restart: unless-stopped
    depends_on:
      plane-api:
        condition: service_healthy
    command: ["celery", "-A", "plane", "beat", "-l", "info"]
    environment:
      DATABASE_URL: "postgresql://bankadmin:${POSTGRES_PASSWORD}@postgres:5432/plane"
      REDIS_URL: "redis://:${REDIS_PASSWORD}@redis:6379/0"
      SECRET_KEY: "${PLANE_SECRET_KEY}"
    networks:
      - bank-internal

  # -------------------------------------------------------------------------
  # Outline - 知識庫與文件協作平台
  # 取代: 散落的 Word、PDF、紙本文件
  # 功能: Markdown 編輯、全文搜尋、協作、版本歷史
  # -------------------------------------------------------------------------
  outline:
    image: outlinewiki/outline:latest
    container_name: bank-outline
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
      minio:
        condition: service_healthy
    environment:
      # 資料庫連線
      DATABASE_URL: "postgresql://bankadmin:${POSTGRES_PASSWORD}@postgres:5432/outline"
      REDIS_URL: "redis://:${REDIS_PASSWORD}@redis:6379/1"
      # 應用設定
      SECRET_KEY: "${OUTLINE_SECRET_KEY}"
      UTILS_SECRET: "${OUTLINE_UTILS_SECRET}"
      URL: "http://outline.bank.local"
      PORT: 3001
      # 檔案儲存 (MinIO S3 相容)
      FILE_STORAGE: "s3"
      AWS_ACCESS_KEY_ID: "${MINIO_ACCESS_KEY}"
      AWS_SECRET_ACCESS_KEY: "${MINIO_SECRET_KEY}"
      AWS_REGION: "us-east-1"
      AWS_S3_UPLOAD_BUCKET_NAME: "outline-data"
      AWS_S3_UPLOAD_BUCKET_URL: "http://minio:9000"
      AWS_S3_FORCE_PATH_STYLE: "true"
      AWS_S3_ACL: "private"
      # 封閉網路 - 停用外部功能
      ENABLE_UPDATES: "false"
      WEB_CONCURRENCY: 2
    ports:
      - "3001:3001"
    networks:
      - bank-internal

  # -------------------------------------------------------------------------
  # Homepage - 統一入口儀表板
  # 用途: 所有內部系統的單一入口，顯示各服務狀態
  # 設定方式: 透過 YAML 檔案設定 (見 ./homepage/ 目錄)
  # -------------------------------------------------------------------------
  homepage:
    image: ghcr.io/gethomepage/homepage:latest
    container_name: bank-homepage
    restart: unless-stopped
    volumes:
      # Homepage 設定檔目錄
      - ./homepage/config:/app/config
      # 讓 Homepage 可以讀取 Docker 狀態
      - /var/run/docker.sock:/var/run/docker.sock:ro
    ports:
      - "3002:3000"
    networks:
      - bank-internal
    environment:
      PUID: 1000
      PGID: 1000

  # =========================================================================
  # 反向代理 (Reverse Proxy)
  # =========================================================================

  # -------------------------------------------------------------------------
  # Nginx - 反向代理與 SSL 終端
  # 用途: 統一管理所有服務的路由與 HTTPS
  # -------------------------------------------------------------------------
  nginx:
    image: nginx:alpine
    container_name: bank-nginx
    restart: unless-stopped
    depends_on:
      - plane-web
      - outline
      - homepage
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      # 自簽 SSL 憑證（內網使用）
      - ./nginx/ssl:/etc/nginx/ssl:ro
    ports:
      - "80:80"
      - "443:443"
    networks:
      - bank-internal
```
